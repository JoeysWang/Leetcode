## 第 0 步：先搞清楚 DP 到底在干什么

### 从一个问题开始：斐波那契数列

斐波那契定义：`fib(1)=1, fib(2)=1, fib(n)=fib(n-1)+fib(n-2)`。

你想要 `fib(5)`。最直觉的写法是递归：

```java
int fib(int n) {
    if (n <= 2) return 1;
    return fib(n-1) + fib(n-2);
}
```

看起来很自然，但画出递归树你会发现一个**致命问题——重复计算**：

```
                fib(5)
              /        \
          fib(4)       fib(3)        ← fib(3) 这里算了一次
         /      \      /     \
     fib(3)   fib(2) fib(2) fib(1)   ← fib(3) 这里又算了一次！
    /     \
 fib(2) fib(1)
```

数一下：`fib(3)` 被算了 **2 次**，`fib(2)` 被算了 **3 次**。n 越大，重复越恐怖——`fib(50)` 这样算能算到你退休。

### 第一个救场思路：记忆化（把算过的存起来）

既然重复算的根源是"忘了自己算过"，那就**记下来**。搞一个备忘录（数组或 Map），算过的值存进去，下次要再用直接查：

```java
int[] memo = new int[n+1];   // memo[i] 记录 fib(i) 的结果，0 表示没算过
int fib(int n) {
    if (n <= 2) return 1;
    if (memo[n] != 0) return memo[n];          // 算过就直接返回
    memo[n] = fib(n-1) + fib(n-2);             // 算完存起来
    return memo[n];
}
```

这下 `fib(3)` 只算一次，再要它就直接查表。复杂度从指数级降到了 O(n)。

**这就是动态规划的雏形**——"把重叠子问题的结果存起来，避免重复计算"。

### 第二个思路：反过来算（自底向上）

递归是"从 fib(5) 往下要 fib(4)、fib(3)…"，是**自顶向下**。
既然 `fib(3)` 依赖 `fib(2)` 和 `fib(1)`，我能不能反过来，**先算小的，再算大的**？

```java
int fib(int n) {
    int[] dp = new int[n+1];
    dp[1] = 1; dp[2] = 1;            // 先把最小的填好
    for (int i = 3; i <= n; i++)      // 从小到大填
        dp[i] = dp[i-1] + dp[i-2];
    return dp[n];
}
```

求 `fib(5)` 时数组一步步变化：

```
初始:        dp = [_, 1, 1, 0, 0, 0]   （下标 0..5，0 暂时没用）
i=3: dp[3]=dp[2]+dp[1]=1+1=2 →  [_, 1, 1, 2, 0, 0]
i=4: dp[4]=dp[3]+dp[2]=2+1=3 →  [_, 1, 1, 2, 3, 0]
i=5: dp[5]=dp[4]+dp[3]=3+2=5 →  [_, 1, 1, 2, 3, 5]
```

返回 `dp[5]=5`。✅ 没有递归、没有重复，一个循环搞定。

**这就是标准的动态规划**——自底向上、从小到大、用前面算好的值推出后面的值。

> 💡 **DP 的本质一句话**：发现大问题能拆成小问题，且小问题会被反复用到，于是**先算小、存起来、再算大**。存的那个数组就叫 **dp 表**。

## 第 1 步：什么时候用 DP

看到题目有这些特征，就往 DP 想：

1. **求最值**：最长 / 最短 / 最多 / 最少 / 最大路径和…
2. **求方案数**：有多少种走法 / 多少种组合…
3. **求可行性**：能否到达 / 能否分割…
4. **能拆成子问题**：大问题的答案能用小问题的答案凑出来。
5. **子问题会重复**：朴素递归会反复算同一个子问题。

> ❌ **什么情况不用 DP**：
> - 要找"连续的一段"（子串/子数组）→ 优先想**滑动窗口**或**双指针**。
> - 每一步局部最优就是全局最优 → 想**贪心**。
> - 要列举所有方案（不是计数）→ 想**回溯**。

举例辨析（容易混的）：
- `3 无重复字符的最长子串` → 要"连续" → 滑动窗口，**不是 DP**。
- `1143 最长公共子序列` → 可跳过字符、不连续 → **DP**。
- 两题都叫"最长…子…"，但解法属于不同阵营。判别法：**"连续" → 窗口；"可跳过/可增删" → DP**。

## 第 2 步：DP 五步法（套路）

拿到一道 DP 题，按这个顺序想，几乎不会卡：

```
1. 定义状态     dp[i] 表示什么？（用一句话写死"i 的语义"）
2. 转移方程     dp[i] = ？？（由哪些更小的 dp 推出来）← 灵魂
3. 初始化       最小的那些 dp[?] = ？
4. 遍历顺序     从哪里填到哪里，才能保证要用到的子状态已经算好
5. 空间优化     能否不用整个数组，只留几个变量？（可选）
```

下面用三道真实题目反复演练这个五步法，直到它变成本能。

## 第 3 步：一维 DP 详解（4 道题）

### 题 A：爬楼梯（70）——五步法入门

**题意**：n 阶楼梯，每次能爬 1 或 2 阶，爬到顶有多少种方法？

**走五步：**

**① 定义状态**：`dp[i]` = 爬到第 i 阶的方法数。

**② 转移方程**：到第 i 阶，上一步要么从 i-1 爬 1 阶来，要么从 i-2 爬 2 阶来。所以：
```
dp[i] = dp[i-1] + dp[i-2]
```
（把"到达 i-1 的方法数"和"到达 i-2 的方法数"加起来，因为最后一步不同，不会重复。）

**③ 初始化**：`dp[1]=1`（爬 1 阶，1 种），`dp[2]=2`（爬 2 阶：1+1 或 2，2 种）。

**④ 遍历顺序**：从小到大，i 从 3 到 n。

**⑤ 空间优化**：只用得到前两个，留两个变量就够。

**手算 n=5 的 dp 表**：

```
i=1: dp[1]=1
i=2: dp[2]=2
i=3: dp[3]=dp[2]+dp[1]=2+1=3
i=4: dp[4]=dp[3]+dp[2]=3+2=5
i=5: dp[5]=dp[4]+dp[3]=5+3=8
```

答案 8 种。✅（你会发现这其实就是斐波那契，只是起点不同。）

**空间优化版**（只留前两个变量）：

```java
public int climbStairs(int n) {
    if (n <= 2) return n;
    int prev = 1, cur = 2;            // prev=dp[1], cur=dp[2]
    for (int i = 3; i <= n; i++) {
        int next = prev + cur;        // dp[i] = dp[i-2] + dp[i-1]
        prev = cur;                   // 整体右移
        cur = next;
    }
    return cur;
}
```
n=5 时变量变化：`(prev=1,cur=2)` → i=3: next=3, `(1→2, 2→3)` → i=4: next=5, `(2,3→5)` → i=5: next=8, `(3,5→8)`。返回 8。✅

### 题 B：打家劫舍（198）——"选/不选"决策型 DP

**题意**：一排房屋，每间有金额 `nums[i]`，相邻两间不能都偷。求最大金额。

**走五步：**

**① 定义状态**：`dp[i]` = 考虑前 i+1 间房（下标 0..i）能偷到的最大金额。

**② 转移方程**：对第 i 间房，只有两种选择：
- **不偷它** → 收益 = `dp[i-1]`（和前 i 间的最优一样）
- **偷它** → 第 i-1 间不能偷，收益 = `dp[i-2] + nums[i]`
取较大：
```
dp[i] = max(dp[i-1],  dp[i-2] + nums[i])
         不偷          偷
```

**③ 初始化**：
- `dp[0] = nums[0]`（只有一间，偷）
- `dp[1] = max(nums[0], nums[1])`（两间选大的）

**④ 遍历顺序**：从左到右，i 从 2 到 n-1。

**⑤ 空间优化**：只依赖前两个，可滚动。

**手算 `nums=[1,2,3,1]`**：

```
dp[0] = nums[0] = 1
dp[1] = max(nums[0], nums[1]) = max(1,2) = 2
dp[2] = max(dp[1],  dp[0]+nums[2]) = max(2, 1+3=4) = 4   ← 偷第3间(值3)更划算
dp[3] = max(dp[2],  dp[1]+nums[3]) = max(4, 2+1=3) = 4   ← 不偷第4间
```

答案 4（偷下标 0 和 2：1+3=4）。✅

**对照你题解里的两种写法**：
- `rob()` = 记忆化递归（自顶向下，带 memo）
- `rob2()` = 自底向上填 dp 表（就是上面这个）

两者用**同一个转移方程** `max(dp[i+1], dp[i+2]+nums[i])`，只是方向相反。看懂一种，另一种也就懂了。

### 题 C：最长上升子序列 LIS（300）——"枚举前驱"型 DP

**题意**：给数组 `nums`，找最长严格递增子序列的长度（子序列可不连续）。

**走五步：**

**① 定义状态**：`dp[i]` = **以 `nums[i] 结尾`** 的最长上升子序列长度。
（⚠️ "以 i 结尾"这个限定很重要，否则无法保证递增关系。）

**② 转移方程**：要看哪些 j < i 且 `nums[j] < nums[i]`，把 `nums[i]` 接到那个序列后面：
```
dp[i] = max( dp[j] + 1 )   对所有 j < i 且 nums[j] < nums[i]
```
若没有这样的 j（i 前面没有比它小的），`dp[i]=1`（自己单独一个）。

**③ 初始化**：每个 `dp[i]` 至少为 1（自己）。

**④ 遍历顺序**：i 从 0 到 n-1；内层 j 从 0 到 i-1。

**⑤ 答案**：`max(dp[i])`（不一定以最后一个结尾）。

**手算 `nums=[10,9,2,5,3,7,101,18]`**，逐个填 dp：

```
i=0, 值10: 前面没有 → dp[0]=1
i=1, 值 9: 看 j=0(10), 10<9? 否 → dp[1]=1
i=2, 值 2: 看 j=0(10)否, j=1(9)否 → dp[2]=1
i=3, 值 5: 看 j=0(10)否, j=1(9)否, j=2(2)是 → dp[3]=dp[2]+1=2
i=4, 值 3: 看 j=0否,j=1否,j=2(2)是→dp[2]+1=2, j=3(5)否 → dp[4]=2
i=5, 值 7: j=0否,j=1否,j=2(2)是→2, j=3(5)是→dp[3]+1=3, j=4(3)是→dp[4]+1=3 → dp[5]=3
i=6, 值101: 所有前面都<101, 取最大 dp[5]+1=3+1=4 → dp[6]=4
i=7, 值18: j=0(10)是→2,j=1(9)是→2,j=2(2)是→2,j=3(5)是→3,j=4(3)是→3,j=5(7)是→4,j=6(101)否 → dp[7]=4
```

dp 数组：`[1,1,1,2,2,3,4,4]`，最大值 4。✅
（一个 LIS 是 `2,5,7,101` 或 `2,3,7,18`，长度都是 4。）

> 这题复杂度 O(n²)。还有 O(n log n) 的"耐心排序+二分"进阶写法，先不展开，把 O(n²) 的 DP 思路吃透再说。

## 第 4 步：二维 DP 详解（2 道题）

一维 DP 是一个下标 `i`；二维 DP 是两个下标 `(i, j)`，dp 表是一张二维表格。典型场景：**两个序列**的问题。

### 题 D：最长公共子序列 LCS（1143）

**题意**：给两个字符串 `s1`、`s2`，求它们的最长公共子序列长度（可不连续）。

**走五步：**

**① 定义状态**：`dp[i][j]` = `s1 的前 i 个字符` 与 `s2 的前 j 个字符` 的最长公共子序列长度。

**② 转移方程**：看 `s1` 的第 i 个字符 `s1[i-1]` 和 `s2` 的第 j 个字符 `s2[j-1]`：
- **相等** → 这一对可以匹配，长度 = `dp[i-1][j-1] + 1`
- **不等** → 这对不能同时匹配，取两边较大的：`max(dp[i-1][j], dp[i][j-1])`
```
if s1[i-1] == s2[j-1]:  dp[i][j] = dp[i-1][j-1] + 1
else:                   dp[i][j] = max(dp[i-1][j], dp[i][j-1])
```

**③ 初始化**：`dp[0][*]=0`（s1 为空，公共长度 0），`dp[*][0]=0`。

**④ 遍历顺序**：i 从 1 到 len1，j 从 1 到 len2（从左上往右下填）。

**⑤ 答案**：`dp[len1][len2]`。

**手算 `s1="abcde"`, `s2="ace"`**（表格，行=i 对应 s1 前 i 个，列=j 对应 s2 前 j 个）：

逐格讲解（只挑几个关键格，其余同理）：

- `(1,1)`：s1[0]='a' vs s2[0]='a'，**相等** → `dp[0][0]+1 = 0+1 = 1`
- `(1,2)`：'a' vs 'c'，不等 → `max(dp[0][2]=0, dp[1][1]=1) = 1`
- `(1,3)`：'a' vs 'e'，不等 → `max(dp[0][3]=0, dp[1][2]=1) = 1`
- `(3,2)`：s1[2]='c' vs s2[1]='c'，**相等** → `dp[2][1]+1 = 1+1 = 2`  ← 关键格
- `(5,3)`：s1[4]='e' vs s2[2]='e'，**相等** → `dp[4][2]+1 = 2+1 = 3`  ← 答案格

完整表：

```
        j=0(空)  j=1(a)  j=2(c)  j=3(e)
i=0(空)    0       0       0       0
i=1(a)     0       1       1       1
i=2(b)     0       1       1       1
i=3(c)     0       1       2       2
i=4(d)     0       1       2       2
i=5(e)     0       1       2       3   ← dp[5][3]=3
```

答案 3，对应 LCS = `"ace"`。✅

> **看懂这张表的窍门**：每个格子的值，要么来自"左上角对角线 +1"（当前两字符相等），要么来自"上方或左方的较大值"（不相等时）。整张表就是这两种操作一格一格填出来的。

### 题 E：编辑距离（72）——二维 DP 的经典（拓展，本目录暂未收录）

**题意**：把 `s1` 变成 `s2`，最少需要多少次操作？操作有三种：插入、删除、替换一个字符。

**走五步：**

**① 定义状态**：`dp[i][j]` = `s1 的前 i 个字符` 变成 `s2 的前 j 个字符` 的最少操作数。

**② 转移方程**：
- 若 `s1[i-1] == s2[j-1]`：这对字符相同，不用操作 → `dp[i-1][j-1]`
- 否则，从三种操作里选最少的 +1：
  - 删除 `s1[i-1]`：`dp[i-1][j] + 1`（s1 少一个，j 不变）
  - 插入（相当于 s2 少一个）：`dp[i][j-1] + 1`
  - 替换：`dp[i-1][j-1] + 1`
```
if 相等:  dp[i][j] = dp[i-1][j-1]
else:     dp[i][j] = 1 + min(dp[i-1][j], dp[i][j-1], dp[i-1][j-1])
```

**③ 初始化**：`dp[i][0]=i`（把 s1 前 i 个全删了），`dp[0][j]=j`（全插入）。

**④ 遍历顺序**：从左上到右下。

**⑤ 答案**：`dp[len1][len2]`。

**手算 `s1="horse"`, `s2="ros"`**：

完整表：

```
        j=0(空)  j=1(r)  j=2(o)  j=3(s)
i=0(空)    0       1       2       3
i=1(h)     1       1       2       3
i=2(o)     2       2       1       2
i=3(r)     3       2       2       2
i=4(s)     4       3       3       2
i=5(e)     5       4       4       3   ← dp[5][3]=3
```

挑几个关键格讲：
- `(1,1)`：'h' vs 'r'，不等 → `1+min(上=1, 左=1, 对角=0) = 1`（替换 h→r）
- `(2,2)`：'o' vs 'o'，**相等** → 直接 `dp[1][1] = 1`（这对白送，不用操作）
- `(3,1)`：'r' vs 'r'，**相等** → `dp[2][0] = 2`
- `(5,3)`：'e' vs 's'，不等 → `1+min(上=dp[4][3]=2, 左=dp[5][2]=4, 对角=dp[4][2]=3) = 1+2 = 3`

答案 3。✅ 一种操作序列：`horse → rorse(替换h为r) → rose(删r) → ros(删e)`，3 步。

> 编辑距离和 LCS 是**同一类**——都是两个序列的二维 DP，转移都依赖"上、左、左上"三个格子。学会一个，另一个就只是方程不同。

## 第 5 步：树形 DP 详解（337 打家劫舍 III）

前面 DP 都是数组/字符串，状态是一维或二维下标。**树形 DP 的状态是"树上的节点"**，遍历方式变成**后序遍历**（先算左右子树，再算当前节点）。

### 题 F：打家劫舍 III（337）

**题意**：房屋排成一棵二叉树，**直接相连的父子**不能同时偷。求最大金额。

**为什么不能用一维/二维 DP？** 因为树不是线性的，没有"下标 i-1"。每个节点的决策依赖它的**子树**，所以要在树上做 DP。

**走五步（思路换成树形版）：**

**① 定义状态**：每个节点返回**两个值**：
- `rob`：偷当前节点时，以当前节点为根的子树能获得的最大金额
- `noRob`：不偷当前节点时，子树的最大金额

**为什么是两个值？** 因为父节点要知道"这个子节点有没有被偷"才能决定自己能不能偷。一个值不够用——这正是你之前在 124 题里纠结的"返回一个还是两个"的同款问题。

**② 转移方程**：
- **偷当前** → 左右子节点都不能偷：`val + 左.noRob + 右.noRob`
- **不偷当前** → 左右子节点各自抢不抢都行，取各自最优：`max(左.rob, 左.noRob) + max(右.rob, 右.noRob)`

**③ 初始化**：空节点返回 `{0, 0}`。

**④ 遍历顺序**：后序（左 → 右 → 根）。

**⑤ 答案**：根节点返回后取 `max(根.rob, 根.noRob)`。

**手算示例树**（题解里的用例 1）：

```
        3          ← 根
       / \
      2   3
       \   \
        3   1
```
（叶子是左下角的 3 和右下角的 1。）

**后序：先算叶子，再往上。** 每个节点算出 `{rob, noRob}`：

```
叶子 3 (左下): rob = 3 + 0 + 0 = 3,  noRob = 0+0 = 0        → {3, 0}
叶子 1 (右下): rob = 1 + 0 + 0 = 1,  noRob = 0+0 = 0        → {1, 0}

节点 2 (val=2): 左子空{0,0}, 右子=叶子3{3,0}
  rob   = 2 + 左.noRob(0) + 右.noRob(0) = 2
  noRob = max(左.rob,左.noRob)(=0) + max(右.rob,右.noRob)(=3) = 3
  → {2, 3}

节点 3 (val=3, 右子树): 左子空{0,0}, 右子=叶子1{1,0}
  rob   = 3 + 0 + 0 = 3
  noRob = 0 + max(1,0) = 1
  → {3, 1}

根 3 (val=3): 左={2,3}, 右={3,1}
  rob   = 3 + 左.noRob(3) + 右.noRob(1) = 3+3+1 = 7
  noRob = max(2,3) + max(3,1) = 3 + 3 = 6
  → {7, 6}
```

答案 `max(7, 6) = 7`。✅（偷根 3、左下叶子 3、右下叶子 1 = 7，根和叶子不相邻，合法。）

> 你题解里用 `memo + dp(root)` 返回单个值、内部比较 rob/noRob 的写法，本质相同——只是把"两个状态"折叠进了 memo 这个全局容器。返回 `{rob, noRob}` 元组的写法更清晰、无副作用，见 124 题我们讨论过的"方法二"。

## 第 6 步：自顶向下 vs 自底向上（同一道题两种写法）

| | 自顶向下 | 自底向上 |
|---|---|---|
| 实现 | 递归 + memo | 循环填 dp 表 |
| 思维 | 贴近自然（从大问题拆） | 先小后大（从 base 往上填） |
| 栈开销 | 有递归栈 | 无 |
| 空间优化 | 难优化 | 可滚动变量 / 滚动数组 |
| 何时偏好 | 状态空间大但实际用到的少 | 一般情况首选 |

**两者用同一个转移方程，完全等价。** 你的 198 题解正好对照：
- `rob()` 递归 + memo：`dp(start) = max(dp(start+1), dp(start+2)+nums[start])`
- `rob2()` 循环填表：`dp[i] = max(dp[i+1], dp[i+2]+nums[i])`（从后往前填）

理解一种，另一种换个方向就行。

## 第 7 步：常见坑（新手最容易踩的）

1. **状态定义含糊**。`dp[i]` 到底表示"前 i 个"还是"以 i 结尾"？写代码前先用一句话钉死。LIS 必须"以 i 结尾"，否则递增关系接不上。

2. **初始化漏了或错值**。比如 LCS 没把 `dp[0][*]` 和 `dp[*][0]` 设 0，整张表就全错。

3. **遍历顺序反了**。规律：**依赖谁，就后算谁**——要用 `dp[i-1]` 就得先算 `dp[i-1]`，所以 i 从小到大。二维 DP 依赖"上、左、左上"，所以从左上往右下填。

4. **答案取错位置**。不是所有题答案都在 `dp[n]`。LIS 答案是 `max(dp[0..n-1])`，因为最优子序列不一定以最后一个结尾。

5. **混淆 DP 和滑动窗口**。再强调一次判别法：
   - 要"连续段" → 滑动窗口
   - 要"可跳过/可增删" → DP

6. **空间优化时方向错**（0-1 背包降一维必须**倒序**遍历容量 w，否则同一物品会被重复取）。本目录暂无背包题，先留个印象。

## 第 8 步：拓展方向（本目录暂未收录，留坑）

- **区间 DP**：`dp[i][j]` 表示区间 `[i,j]` 上的答案，转移枚举切分点 `dp[i][j] = min(dp[i][k]+dp[k+1][j])`，按**区间长度**从小到大填。例：516 最长回文子序列、312 戳气球。
- **背包 DP**：`dp[i][w]` 表示前 i 个物品、容量 w 下的最优，`dp[i][w] = max(dp[i-1][w], dp[i-1][w-wt[i]]+val[i])`。例：0-1 背包、完全背包、322 零钱兑换。
- **状态压缩 DP**：用二进制位表示集合状态，用于旅行商等问题。

## 总结：DP 解题心法

1. **先判断是不是 DP**（求最值/方案数/可行性 + 能拆子问题 + 子问题重叠）。
2. **五步法**：定义状态 → 转移方程 → 初始化 → 遍历顺序 → 空间优化。
3. **转移方程是灵魂**，其它都是配套。想不出方程，就动手**列小例子的 dp 表**，填着填着规律就出来了。
4. **背不下来就别背**——拿本文的 6 道题，每道亲手填一遍 dp 表（别看答案），填 3 遍自然就通。
